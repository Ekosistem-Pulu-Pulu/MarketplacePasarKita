User Journey Map

Visualisasi alur peran pengguna marketplace berdasarkan implementasi Bagian 6 — dokumentasi spesifikasi PasarKita. Setiap journey memetakan fase, sentuhan/channel, status, dan cabang keputusan yang tersedia di source code.

Catatan: langkah visual pada peta ini merepresentasikan abstraksi alur. Detail mikrokopis UI (banner profil, auto-refresh, dsb.) yang tidak ditemukan secara eksplisit pada source code ditandai dengan istilah "ilustrasi alur" dan bukan jaminan implementasi.

Journey 01

Guest → Buyer

Role: Guest → Buyer (authenticated)
Discovery › Browse › Cart › Auth › Checkout › Payment › Fulfillment
Discovery & Browse
Guest
  • 1
    Buka Landing
    Mengakses beranda, melihat banner promo, kategori, dan produk pilihan.
    Public
    Channel: Home (#/)
  • 2
    Cari / Pilih Kategori
    Mengetik kata kunci di search bar atau memilih salah satu dari 8 kategori.
    Public
    Channel: Search & Category
  • 3
    Telusuri Listing
    Membuka daftar produk aktif, melihat product card (nama, harga, rating, terjual).
    Public
    Filter berdasarkan kategori, harga, rating
  • 4
    Buka Detail Produk
    Membaca deskripsi, galeri, harga, stok, rating, dan garansi retur 7 hari.
    Public
    Menampilkan informasi toko & ulasan
Keranjang & Autentikasi
Auth Gate
  • 5
    Tambah ke Keranjang
    Klik "Tambah ke Keranjang" dari ProductCard atau detail. Item masuk ke localStorage / API.
    Trigger
    Sync cart guest ↔ user saat login
  • 6
    Login atau Register
    Checkout memicu login modal. User memasukkan email + password atau membuat akun baru.
    Required
    Validasi: 8+ karakter, ada huruf besar, kecil, angka, simbol
    Channel: Login / Register / Forgot Password
Cabang Otentikasi
  • Login Berhasil → JWT token tersimpan, kembali ke cart
  • Akun Baru → Validasi register (email unik, password strength)
  • Lupa Password → Forgot Password flow
Checkout & Fulfillment
Buyer
  • 7
    Tinjau Keranjang
    Memperbarui qty, menghapus item, menerapkan voucher (PERCENT/AMOUNT).
    Buyer
    Voucher valid + diskon otomatis subtotal ≥ ambang
  • 8
    Checkout
    Sistem menghitung subtotal + marketplace fee (2%) + ongkir (kosong) + total.
    Calculated
    Validasi qty ≤ stok, voucher aktif
  • 9
    Pilih Metode Pembayaran
    Snap pembayaran atau Virtual Account. Sistem mengirim payment request.
    Pending
    Channel: Payment (#/payment/:id)
  • 10
    Status Order: Pending → Paid
    Order dibuat dengan status PENDING_PAYMENT, lalu menunggu hasil SmartBank.
    Pending
    External: Tergantung callback SmartBank
  • 11
    Processing → Shipped → Delivered
    Seller memproses, mengirim barang. User melihat status progresif di OrderStepper.
    Active
    Auto-refresh status order setelah pembayaran sukses
  • 12
    Selesaikan & Review
    Pesanan diterima (Completed). Opsional: tinggalkan review & rating produk.
    Done
    Garansi retur 7 hari aktif sejak diterima
Outcome Pembayaran (system-driven)
  • Pembayaran Gagal → Order tetap tersimpan, beri retry
  • Stok Habis Saat Checkout → Tampilkan error validasi
Status warna: Discovery / Eksplorasi Trigger / Aksi Validasi / Pending Authenticated Step Berhasil / Selesai Validasi Gagal
Journey 02

Non-Seller → Seller Application → Seller

Role: Buyer → Applicant → Active Seller
Profile › Application › Review › Approval › Onboarding › Active Selling
Profil & Aplikasi
Buyer
  • 1
    Buka Halaman Profil
    User login lalu navigasi ke menu Profile. Melihat identitas, voucher, dan status keanggotaan.
    Buyer
    Channel: Profile (#/profile)
  • 2
    Pilih Menu "Daftar Jadi Penjual"
    Dari profil, klik opsi Ajukan Aplikasi Seller. Sistem memuat form aplikasi.
    Buyer
    Motivasi: menambah penghasilan, memperluas jangkauan
  • 3
    Isi Form Aplikasi
    Nama toko, deskripsi, kategori usaha, informasi pemilik, kontak, alamat.
    Form
    Validasi real-time dengan Zod
  • 4
    Validasi & Submit
    Sistem mengecek kelengkapan field dan mengirim aplikasi ke tim internal marketplace.
    Submit
    Kalau ada field kosong atau salah format → tampilkan error
Proses Review
Applicant
  • 5
    Aplikasi Terkirim (Pending)
    Status berubah menjadi "Diajukan" / Pending. User melihat banner info di profil.
    Pending
    User mendapat kepastian aplikasi sudah masuk antrian
  • 6
    Verifikasi Tim Internal
    Catalog admin / customer support mengecek data toko, kontak, dan kelengkapan dokumen.
    In-Review
    Dilakukan oleh peran catalog_admin dan customer_support
  • 7
    Keputusan Review
    Tim memutuskan menyetujui (approve) atau menolak aplikasi dengan alasan.
    Decision
    Dilakukan oleh catalog_admin
Cabang Keputusan Review
  • Disetujui → status berubah menjadi ACTIVE
  • Ditolak → tampilkan alasan, user boleh mengajukan ulang
Aktif sebagai Penjual
Seller
  • 8
    Aktivasi Seller
    Role user berubah menjadi seller. Menu seller dashboard muncul otomatis.
    Active
    Akses Seller Dashboard terbuka
  • 9
    Kelola Produk (CRUD)
    Tambah, ubah, nonaktifkan produk dengan SellerTable sebagai daftar transaksi.
    Owned
    Seller tidak boleh mengubah produk seller lain
  • 10
    Lihat Order Masuk
    Order baru tampil di SellerDashboard dengan status pembayaran dan info pembeli.
    Order
    Channel: Seller Dashboard Stats & Seller Table
  • 11
    Proses & Kirim Pesanan
    Order berpindah dari PAID → PROCESSING → SHIPPED. Sistem memicu pengiriman logistik.
    Process
    Update status order = audit log tercatat
  • 12
    Selesaikan & Terima Pembayaran
    Setelah pesanan diterima, dana diteruskan dari SmartBank ke seller sesuai aturan fee 2%.
    Settled
    SellerStats menampilkan metrik penjualan
Catatan Operasional
  • Stok habis → produk otomatis nonaktif di browse publik
  • Order dibatalkan → marketplace fee tidak dipotong
Status warna: Discovery / Keputusan Form / Aksi Verifikasi / Pending Seller Aktif Review Internal Berhasil / Selesai
